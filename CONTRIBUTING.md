# Contributing to SUSTAINABILITY.md

Thanks for helping improve this project.

## Project status

This project is in draft and evolves through practical use, review, and iteration.

## AI disclosure

This repository uses AI-assisted drafting and implementation support. Human contributors are responsible for reviewing, correcting, and approving changes.

## What to contribute

- Improvements to policy clarity and actionable guidance.
- Better mappings between policy and WSG references.
- Reductions in compute, transfer size, and third-party impact.
- CI guardrails that improve measurability.
- Documentation that helps teams apply sustainability practices day-to-day.

## Contribution principles

- Prefer measurable improvements over broad claims.
- Keep proposals practical and implementable.
- Explain trade-offs clearly (performance, UX, business, and impact).
- Favor progress over perfection.

## How to contribute

1. Open an issue describing the problem or proposal.
2. Submit a pull request with focused changes.
3. Include before/after impact notes when possible.
4. Link relevant WSG references for policy or mapping updates.

## Pull request checklist

- Scope is clear and minimal.
- Language is specific and action-oriented.
- Any automation changes are explained.
- Documentation is updated if behavior or policy changes.

## Code of collaboration

Be constructive, respectful, and evidence-based. Good-faith participation from maintainers, contributors, and reviewers is expected.
